알파인 리눅스에서 타임존 설정하기 : 로컬 환경과 도커 이미지에서의 처리 방법

프로덕션 환경에서는 cron 작업이나 로그 시간 현지화 그리고 DB와 OS 간의 시간 동기화 등의 이유로 직접 타임존을 설정해야 하는 경우가 있다. 타임존 설정 방법은 리눅스 배포판 별로 차이가 있는데, 이번 글에서는 경량 리눅스 배포판 중 하나인 Alpine linux에서의 타임존 설정 방법을 알아보고, 도커 환경에서 설정할 시 최적화하여 적용하는 방법에 대해서도 알아보겠다.

로컬 환경에서의 타임존 설정

먼저, 직접 컴퓨터에 설치해서 사용하는 로컬 환경에서의 설정 방법을 알아보겠다.

tzdata 패키지 설치하기

알파인 리눅스에는 타임존 설정 관련 패키지가 포함되어 있지 않으므로, tzdata 패키지를 설치하도록 한다.

Terminal
apk add tzdata

패키지가 정상적으로 설치가 되었는지 ls -L 명령어로 확인해본다. 이 명령어는 심볼릭 링크가 가리키는 실제 파일이나 디렉토리의 정보를 표시한다.

ls -L /usr/share/zoneinfo/

localtime 파일에 원하는 타임존으로 심볼릭 링크 설정

필자는 한국 시간으로 타임존을 설정할 것이기 때문에 Asia/Seoul 파일에 심볼릭 링크를 설정할 것이다. 심볼릭 링크의 타겟 경로는 /etc/localtime이다.

Terminal
ln -s /usr/share/zoneinfo/Asia/Seoul /etc/localtime

(선택사항) timezone 파일에 명시적으로 타임존 지정

/etc/timezone에 설정한 타임존이 반영되지는 않지만, 명시적인 이용을 위해서 지정해주는 것이 좋다.

Terminal
echo "Asia/Seoul" > /etc/timezone

date 명령어로 확인

date

도커 환경에서의 타임존 설정

도커 환경에서 알파인 리눅스 타임존을 설정할 때는, 위에서 살펴본 각 단계를 그대로 도커 파일로 작성하면 된다. 프로젝트 환경에 따라 선택할 수 있는 단일 스테이지 빌드와 다중 스테이지 빌드를 통해 도커 환경에서의 타임존 설정을 최적화할 수 있다.

단일 스테이지 빌드

단일 스테이지 빌드는 단순한 환경에서 사용되며, 각 단계를 순차적으로 진행하여 이미지를 빌드한다. 다음은 단일 스테이지 빌드를 사용하여 도커 이미지를 빌드하는 방법이다.

Dockerfile
FROM node:18.16.0-alpine3.17  

RUN apk add --no-cache tzdata && \
    cp /usr/share/zoneinfo/Asia/Seoul /etc/localtime && \
    echo "Asia/Seoul" > /etc/timezone && \
    apk del tzdata
  • 3행: --no-cache 플래그가 사용됐는데, 패키지를 설치할 때 캐시를 저장하지 않음으로써, 이미지를 경량화 하기 위해 사용된다.
  • 4행: tzdata 패키지를 삭제할 것이기 때문에, 심볼릭 링크를 설정하지 않고 복사한다.
  • 6행: 더 이상 필요치 않은 tzdata 패키지를 삭제하여 이미지를 경량화 해준다.

다중 스테이지 빌드

Multi-stage builds란?

Docker v17.06부터 지원되는 기능으로, 도커 이미지를 빌드하는 과정에는 필요하지만 최종 결과물에는 필요치 않은 요소들을 쉽게 제외하기 위해서 단계를 나누는 기능이다.

다중 스테이지 빌드는 복잡한 환경에서 강점을 가지며, 빌드 프로세스를 여러 단계로 나누어 최종 이미지에 필요한 파일만 포함시킨다. 다음은 다중 스테이지 빌드를 사용하여 도커 이미지를 빌드하는 방법이다.

Dockerfile
FROM node:18.16.0-alpine3.17 AS base

# Builder process
FROM base AS builder

RUN apk add tzdata

# Runner process
FROM base AS runner

COPY --from=builder /usr/share/zoneinfo/Asia/Seoul /etc/localtime
RUN echo "Asia/Seoul" > /etc/timezone
